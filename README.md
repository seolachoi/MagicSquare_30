# Prompting_BP

Prompting 실습 프로젝트의 회고, 개선 계획, 실행 추적을 위한 문서형 저장소입니다.

## 프로젝트 개요

- 목적: 실습 전 과정을 돌아보고 KPT 기반 회고와 개선 액션을 문서화
- 산출물: 회고 세션 가이드, 액션 아이템 추적 템플릿, 학습 자료 정리, 다음 프로젝트 개선 계획
- 제출 형태: Markdown 문서

## 권장 브랜치 전략

```bash
git checkout develop
git pull origin develop
git checkout -b bp/prompting_bp
```

## 폴더 구조

- `docs/RETROSPECTIVE.md`: KPT 회고 본문 (Keep / Problem / Try)
- `docs/SESSION_GUIDE.md`: 회고 세션 진행 가이드
- `docs/LEARNING_OUTCOMES.md`: 개인별 학습 성과 정리
- `docs/NEXT_PROJECT_PLAN.md`: 현업 적용 및 다음 프로젝트 개선 계획
- `templates/ACTION_ITEMS_TEMPLATE.md`: 팀 개선 액션 아이템 추적 템플릿

## 사용 순서

1. `docs/SESSION_GUIDE.md`로 회고 세션 진행
2. `docs/RETROSPECTIVE.md`에 팀 회고 기록
3. `docs/LEARNING_OUTCOMES.md`에 개인 학습 성과 정리
4. `templates/ACTION_ITEMS_TEMPLATE.md`를 복사해 액션 등록 및 추적
5. `docs/NEXT_PROJECT_PLAN.md`에 다음 프로젝트 실행 계획 확정
