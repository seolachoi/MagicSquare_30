# Prompting_BP 회고 세션 진행 가이드

## 1) 세션 목표

- 실습 과정에서 효과가 있었던 방식과 병목 요인을 명확히 정리한다.
- 다음 실습/프로젝트에 바로 적용할 개선 액션을 합의한다.
- 개인 학습 성과를 팀 자산으로 전환한다.

## 2) 권장 진행 시간 (총 90분)

- 오프닝 및 목표 정렬: 10분
- KPT 작성(개인): 15분
- KPT 공유 및 토론(팀): 30분
- 액션 아이템 합의: 20분
- 우선순위/담당자/일정 확정: 10분
- 마무리: 5분

## 3) KPT 진행 질문

### Keep

- 무엇이 잘 되었는가?
- 어떤 프롬프트/협업 방식이 생산성을 높였는가?
- 다음 프로젝트에서도 반드시 유지할 관행은 무엇인가?

### Problem

- 어디에서 시간이 가장 많이 소모되었는가?
- 요구사항 해석, 테스트, 문서화 중 병목은 무엇이었는가?
- AI 결과 검증 과정에서 반복된 실수는 무엇이었는가?

### Try

- 다음에는 무엇을 다르게 시도할 것인가?
- 어떤 기준/체크리스트를 추가하면 재발을 줄일 수 있는가?
- 실험 가능한 개선안 1~2주는 무엇인가?

## 4) 산출물 체크리스트

- [ ] `docs/RETROSPECTIVE.md` 업데이트
- [ ] 액션 아이템 템플릿 작성 (`templates/ACTION_ITEMS_TEMPLATE.md`)
- [ ] 개인 학습 성과 정리 (`docs/LEARNING_OUTCOMES.md`)
- [ ] 다음 프로젝트 계획 정리 (`docs/NEXT_PROJECT_PLAN.md`)

## 5) 운영 규칙

- 문제 제기 시 사례와 근거를 함께 제시한다.
- 개선안은 "누가, 언제까지, 무엇을, 어떻게 검증할지"까지 확정한다.
- 모호한 표현("열심히", "가능하면") 대신 측정 가능한 문장으로 기록한다.
