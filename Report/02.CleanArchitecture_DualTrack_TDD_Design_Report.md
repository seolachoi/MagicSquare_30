# Report 02 — Clean Architecture & Dual-Track TDD Design

## 1. 고정 계약

| 항목 | 규칙 |
|------|------|
| 입력 | `4×4 int[][]`, `0`=빈칸(정확히 2), 값 `0` 또는 `1~16`, 0 제외 중복 금지 |
| 첫 빈칸 | row-major 첫 `0` |
| 출력 | `int[6]` = `[r1,c1,n1,r2,c2,n2]`, **1-index** |
| 시도 | 작은 누락수→첫 빈칸, 큰→둘째; 실패 시 역순 |
| Magic Constant | `34` (`MagicConstant` SSOT) |

## 2. ECB 레이어

| Layer | 책임 | 컴포넌트 |
|-------|------|----------|
| Boundary | 입력 검증, Failure/OK envelope, UI | `InputValidator`, `UIBoundary` |
| Control | 유스케이스 오케스트레이션 | `SolvePartialMagicSquare` |
| Entity | 순수 도메인 | `EmptyCellLocator`, `MissingNumberFinder`, `MagicSquareValidator`, `TwoCellSolver` |

의존: Boundary → Control → Entity (역방향 금지)

## 3. Dual-Track TDD

| Track | 테스트 | 보호 대상 |
|-------|--------|-----------|
| A (Boundary) | `tests/boundary/` | E001~E005, U-FLOW-02, int[6] 형식 |
| B (Logic) | `tests/entity/` | I1~I11, D-LOC/MIS/VAL/SOL |
